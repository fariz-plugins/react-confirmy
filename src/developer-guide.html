React Confirmy - Developer Guide

├── Icons/        # SVG icons and related components
├── styles/       # Style configurations

Key Components

1. Confirmy Component

The main dialog component that renders the confirmation modal.

// Usage
<Confirmy
    isOpen={isOpen}
    onClose={() => setIsOpen(false)}
    onConfirm={handleConfirm}
    title="Confirm Action"
    message="Are you sure?"
    position="bottom"  // 'top' | 'bottom' | 'center'
    type="warning"     // 'warning' | 'danger' | 'info'
    size="md"         // 'sm' | 'md' | 'lg'
/>

2. DialogProvider

Context provider that manages the state of multiple dialogs.

// Wrap your app
<DialogProvider>
    <App />
</DialogProvider>

3. useConfirmy Hook

Custom hook for managing dialog state and interactions.

Customization

Styling

The plugin supports two styling frameworks:

  • tailwind (default)
  • custom - Use your own CSS
Note: When using custom styles, provide them through the styles prop: 
<Confirmy
    styles={{
        container: 'your-container-class',
        title: 'your-title-class',
        // ...other style overrides
    }}
/>

Form Fields

The dialog supports dynamic form fields:

const formFields = [
    {
        name: 'reason',
        label: 'Reason',
        type: 'text',
        required: true
    },
    {
        name: 'description',
        label: 'Description',
        type: 'textarea'
    }
];

Animation

Customize animations using the animation prop:

<Confirmy
    animation={{
        type: 'scale',
        duration: 200,
        timing: 'ease-out'
    }}
/>

Best Practices

  • Always wrap your application with DialogProvider
  • Use the useConfirmy hook for complex dialog interactions
  • Provide proper ARIA labels for accessibility
  • Use TypeScript types for better development experience
  • Test dialog interactions thoroughly

Type System

Key TypeScript types:

type DialogType = 'warning' | 'danger' | 'info';
type DialogSize = 'sm' | 'md' | 'lg';
type DialogPosition = 'top' | 'bottom' | 'center';
type Framework = 'tailwind' | 'custom';

Troubleshooting

  • If context errors occur, ensure DialogProvider is properly set up
  • For styling issues, check framework prop and style overrides
  • For form validation, ensure required fields are properly marked
